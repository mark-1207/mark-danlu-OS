# 模板格式规范化设计方案

**最后更新**: 2026-03-26
**状态**: 设计中

---

## 1. 背景与目标

### 问题现状
- 提示词硬编码在 TypeScript 代码中
- 用户添加自定义模板时无格式验证
- 粘贴他人提示词时格式混乱，无法直接使用

### 目标
- 定义各类型模板的格式规范
- 实现自动修正功能（类似 Word 的拼写检查+自动修正）
- 用户只需粘贴内容，系统自动整理成规范格式

### 用户场景
主要用户：非程序员，粘贴他人提示词进行微调使用

---

## 2. 格式规范定义

### 2.1 内容模板结构

一份完整的内容模板由两部分组成：

```
# 标题提示词
[标题生成的指令内容]

# 正文提示词
[正文生成的指令内容]
```

### 2.2 标题提示词规范

**必须包含**：
- 角色定义（你是一个XXX）
- 输入变量：`{content}` 或 `{title}`
- 输出格式要求（JSON格式）

**可选包含**：
- 公式/模板说明（"核心公式：颠覆认知型..."）
- 禁止事项（"❌ 禁止..."）
- 字数限制

**示例**：
```markdown
# 标题提示词
你是一个公众号爆款标题专家，根据用户提供的素材生成爆款标题。

【输入】{content}
【受众】{audience}
【关键词】{keywords}

## 输出格式
严格按以下JSON格式输出：
{
  "titles": [
    { "text": "标题", "type": "公式类型", "reason": "原因" }
  ],
  "recommended": "主推标题"
}
```

### 2.3 正文提示词规范

**必须包含**：
- 角色定义
- 输入变量：`{content}`, `{title}`, `{audience}`
- 内容类型说明（访谈/故事/观点/新闻/演讲）

**可选包含**：
- 结构框架（开头→主体→结尾）
- 金句策略
- 禁止事项
- 排版要求

**示例**：
```markdown
# 正文提示词
你是一个公众号内容创作专家，根据用户提供的素材生成文章。

【输入】{content}
【标题】{title}
【受众】{audience}

## 内容类型
根据素材类型调整写作侧重点：
- 访谈对话类：提取核心观点、金句
- 故事分享类：情节线、情感曲线

## 禁止事项
❌ 空洞说教，无具体案例
❌ 过度营销感
```

### 2.4 变量标准列表

系统预设的标准变量：

| 变量 | 含义 |
|------|------|
| `{content}` | 原始内容/素材 |
| `{title}` | 选定标题 |
| `{audience}` | 目标受众 |
| `{keywords}` | 关键词 |
| `{emotion}` | 情绪基调 |
| `{category}` | 内容分类 |
| `{style}` | 风格要求 |
| `{word_count}` | 目标字数 |

## 3. 自动修正逻辑

### 3.1 修正流程

```
用户粘贴文本
    ↓
系统分析文本结构
    ↓
识别各部分内容类型
    ↓
提取和标准化变量
    ↓
整理成规范格式
    ↓
展示修正预览
    ↓
用户确认保存
```

### 3.2 智能识别规则

#### 标题提示词识别
- 包含"标题"、"title"、"生成标题" → 归类为标题提示词
- 包含 JSON 输出格式定义 → 识别为标题提示词
- 包含"5个标题"、"10个标题" → 识别为标题提示词

#### 正文提示词识别
- 包含"正文"、"content"、"生成正文"、"写文章" → 归类为正文提示词
- 包含分段结构（开头、主体、结尾） → 识别为正文提示词
- 包含"2000字"、"3000字"等字数要求 → 识别为正文提示词

#### 变量自动提取
- 原文中的变量占位符（`{content}`, `{原文}` 等）→ 转换为标准变量
- 检测到新变量时提示用户

#### 映射规则（部分）
| 原文写法 | 标准变量 |
|---------|---------|
| 原文、内容、素材 | `{content}` |
| 标题、主标题 | `{title}` |
| 受众、目标人群、读者 | `{audience}` |
| 关键词、核心关键词 | `{keywords}` |

### 3.3 格式整理规则

- 自动添加 `# 标题提示词` / `# 正文提示词` 分隔标记
- 统一使用 `【】` 标记变量
- 提取并格式化 JSON 输出要求
- 清理多余的空行和格式混乱

## 4. 界面交互设计

### 4.1 编辑弹窗改造

**现有流程**：
1. 用户点击"编辑"
2. 弹出文本框显示合并内容
3. 用户手动编辑
4. 点击保存

**改造后流程**：
1. 用户点击"编辑"
2. 弹出文本框
3. **用户粘贴内容**
4. **系统自动分析并显示修正预览**
5. 用户点击"确认修正"或"直接保存"
6. 保存

### 4.2 修正预览展示

```
┌─────────────────────────────────────────┐
│  📋 检测到以下结构                        │
├─────────────────────────────────────────┤
│  ✅ 标题提示词 [已识别]                   │
│     - 包含变量: {content}, {audience}    │
│     - 包含输出格式定义                    │
│                                         │
│  ✅ 正文提示词 [已识别]                   │
│     - 包含内容类型说明                    │
│     - 包含禁止事项                        │
├─────────────────────────────────────────┤
│  ⚠️ 建议修正                              │
│     - 变量 {原文} → 已转换为 {content}   │
│     - 添加了输出格式分隔                  │
│     - 清理了多余空行                      │
├─────────────────────────────────────────┤
│  [查看修正后内容]  [直接保存]  [取消]     │
└─────────────────────────────────────────┘
```

### 4.3 修正详情展开

点击"查看修正后内容"后，显示完整的修正后模板：

```
# 标题提示词
你是一个公众号爆款标题专家...

[修正后的标题内容]

---

# 正文提示词
你是一个公众号内容创作专家...

[修正后的正文内容]
```

## 5. 技术实现

### 5.1 文件结构

```
src/
  data/
    templateSpecs.ts        # 格式规范定义
    index.ts                # 统一导出
  services/
    validator/
      TemplateValidator.ts  # 验证+修正逻辑
      index.ts
  components/
    SettingsPage.tsx        # 改造现有编辑弹窗
```

### 5.2 核心函数

#### `analyzeTemplateContent(text: string): AnalysisResult`
分析用户输入的文本，返回结构化分析结果。

#### `autoFixTemplate(text: string, analysis: AnalysisResult): FixedTemplate`
根据分析结果，自动修正文本格式。

#### `validateTemplate(template: ContentTemplate): ValidationResult`
验证修正后的模板是否符合规范。

### 5.3 类型定义

```typescript
interface AnalysisResult {
  hasTitleSection: boolean;
  hasContentSection: boolean;
  detectedVariables: string[];
  missingVariables: string[];
  structureIssues: string[];
  rawTitleContent: string;
  rawContentContent: string;
}

interface FixedTemplate {
  titlePrompt: string;
  contentPrompt: string;
  appliedFixes: string[];
  warnings: string[];
}
```

## 8. 设计原则

1. **用户只需粘贴**：不需要理解格式规范，系统自动识别
2. **所见即所得**：修正后立即可见，确认后再保存
3. **保留修改权**：用户可选择"直接保存"跳过修正
4. **渐进式**：先支持公众号模板，后续扩展其他平台
