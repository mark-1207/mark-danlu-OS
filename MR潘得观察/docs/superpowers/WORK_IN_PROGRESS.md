## 📌 已完成任务详情

### A) 质检准确性优化 ✅

**改动文件**:
- `src/data/gzhQualityPrompt.ts`
- `src/data/xhsQualityPrompt.ts`
- `src/data/douyinQualityPrompt.ts`
- `src/services/llm/parser/QualityParser.ts`

**解决的核心问题**:
| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| A3 维度定义模糊 | 判定标准只有文字描述，无标杆示例 | 每个维度添加正反例标杆 |
| A1 误判严重 | AI无法精准执行抽象标准 | 评分必须引用原文片段 |
| A4 优化建议不实用 | 输出冗长，缺乏可操作性 | 优化建议改为原文→改后对照表 |
| A2 评分不稳定 | 提示词复杂，AI注意力分散 | 精简输出结构 |

**预期效果**:
- 评分更有依据（必须引用原文）
- 优化建议更可操作
- 输出结构简化，减少AI执行偏差

### E) 模板格式规范化（架构问题）

**优先级**: 高（影响后续所有模板相关工作）

**问题描述**:
当前提示词写在 TypeScript 代码里，不是可动态编辑的配置。用户添加自定义模板时，没有机制确保格式符合要求。

**现状架构**:
```
src/data/gzhContentPrompt.ts  ← 硬编码
         ↓
settingsStore.ts  ← 导入作为内置模板默认值
         ↓
用户自定义模板 → localStorage
```

**待实施方案**:
| 组件 | 作用 | 状态 |
|-----|------|------|
| `src/data/templateSpecs.ts` | 定义各类型模板的格式规范 | 待创建 |
| `TemplateValidator` | 验证自定义模板是否符合规范 | 待创建 |
| 设置页面改造 | 模板编辑时提供格式指引 | 待改造 |
| 模板数据迁移 | 内置模板提示词移出代码 | 待执行 |

**需要做的事情**:

1. **模板数据迁移**
   - [ ] 将内置模板提示词从 TypeScript 代码移到独立数据文件
   - [ ] 保持与现有 settingsStore 的兼容

2. **创建模板格式规范**
   - [ ] 定义 titlePrompt 标准格式要求
   - [ ] 定义 contentPrompt 标准格式要求
   - [ ] 定义 qualityPrompt 标准格式要求
   - [ ] 定义输出格式（JSON结构）规范

3. **实现模板验证器**
   - [ ] 创建 TemplateValidator 函数
   - [ ] 验证自定义模板是否符合规范
   - [ ] 提供格式指引/错误提示

4. **更新设置页面**
   - [ ] 模板编辑时显示格式规范
   - [ ] 提供"基于规范创建"的入口
   - [ ] 用户自定义模板时引导符合格式
