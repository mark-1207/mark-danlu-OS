### A) 质检准确性优化 ✅

- `src/data/xhsQualityPrompt.ts`
- `src/data/douyinQualityPrompt.ts`
- `src/services/llm/parser/QualityParser.ts`

**解决的核心问题**:
| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| A3 维度定义模糊 | 判定标准只有文字描述，无标杆示例 | 每个维度添加正反例标杆 |

| A2 评分不稳定 | 提示词复杂，AI注意力分散 | 精简输出结构 |

**预期效果**:
- 评分更有依据（必须引用原文）
- 优化建议更可操作
- 输出结构简化，减少AI执行偏差

### E) 模板格式规范化（架构问题）🔄

**优先级**: 高（影响后续所有模板相关工作）

**问题描述**:
当前提示词写在 TypeScript 代码里，不是可动态编辑的配置。用户添加自定义模板时，没有机制确保格式符合要求。

**用户需求**:
用户粘贴他人提示词时，系统自动识别结构并修正格式，类似 Word 的拼写检查+自动修正。

**实现方案**:
```
用户粘贴 → 系统分析 → 自动修正 → 一键确认 → 保存
```

**改动文件**:
- `src/data/templateSpecs.ts` ✅
- `src/services/validator/TemplateValidator.ts` ✅
- `src/services/validator/index.ts` ✅
- `src/components/SettingsPage.tsx` ✅

**Phase 1-2 完成**:
| 组件 | 功能 |
|-----|------|
| `templateSpecs.ts` | 定义格式规范、变量标准、别名映射 |
| `TemplateValidator.ts` | analyzeTemplateContent() 分析结构 / autoFixTemplate() 自动修正 |
| `SettingsPage.tsx` | 粘贴自动分析、显示修正预览、一键修正按钮 |

**待完成**:
- [ ] Phase 3: 模板数据迁移（内置提示词从代码移出）
- [ ] 增强变量映射（支持更多别名）
- [ ] 优化修正算法
