# D-003: 4 思想模型框架 + 5 策略模式

## 状态
已敲定 | 2026-06-15

## 背景
思想模型（杠杆者/第一性原理/...）单独用效果有限。需要决定：
- 多少个内置框架
- 框架的策略模式
- 模型在框架里的角色

## 备选方案

### 框架数
| 方案 | 评估 |
|------|------|
| 1 个万能框架 | 等于无框架 |
| 2 个 | 覆盖不全 |
| **4 个** | 拆解/决策/系统/创新 覆盖完整 |
| 8 个 | 用户选择困难 |

### 策略模式
| 模式 | 含义 | 适用 |
|------|------|------|
| chain | A→B→C 链式 | 顺序推导 |
| parallel | A‖B‖C 并行 | 多视角 |
| nested | A(B(C)) 嵌套 | 多层级 |
| divergent→convergent | 发散→收敛 | 新角度 |
| **condition** | **A 或 B 或 C（按条件）** | **内容类型分流** |

## 选定

**4 框架 + 5 策略模式**：

| 框架 | 策略 | 模型 |
|------|------|------|
| 问题解构 | chain | 第一性原理 + 5Why + 奥卡姆剃刀 |
| 决策分析 | parallel | 杠杆者 + JTBD + 反共识 |
| 系统思考 | nested | 系统思维 + 因果链 + 反馈回路 + 二阶效应 |
| 创新突破 | divergent→convergent | 逆向 + 类比 + 第一性原理 + 反共识 |

**v1 跑 5 种策略**，iterate/recursive 留 v2+。

## 后果

### 正面
- 框架有结构，不是物理叠加
- 5 种策略覆盖主要场景
- 模型有"角色"（lead/follow/cleanup），避免堆术语

### 负面
- 实现 5 种 strategy 复杂度高
- 嵌套（nested）容易栈溢出

### 风险
- 嵌套深度失控
- 缓解：递归上限 2 层
