# D-008: 苏格拉底 3 阶段学习机制

## 状态
已敲定 | 2026-06-15

## 背景
苏格拉底追问怎么判断"什么情况算够了"。需要决定：
- 固定轮数 vs 动态判断
- 是否学习用户判断逻辑
- 何时能脱离用户判断

## 备选方案

### A：固定轮数
- 永远 3 轮或 5 轮
- 简单但不灵活

### B：纯 LLM 判
- LLM 决定何时停
- 但 LLM 不可靠

### C：用户说 + 记录
- 靠用户说"够了"
- 记录上下文
- 简单但需用户参与

### D：3 阶段学习（炉选定）
- 阶段 1（0-30 样本）：靠用户说
- 阶段 2（30-100）：系统提示 + 用户决定
- 阶段 3（100+）：自动判断

## 选定
**3 阶段学习**。

理由：
1. 阶段 1 数据积累
2. 阶段 2 半自动
3. 阶段 3 全自动
4. 渐进式，比一步到位更稳

## 3 阶段

| 阶段 | 样本 | 行为 |
|------|------|------|
| 1 | 0-30 | 靠用户说"够了"停 + 记录上下文 |
| 2 | 30-100 | 系统提示"可收尾"，用户决定 |
| 3 | 100+ | 炉 自动判断，不再问 |

## 后果

### 正面
- 学习用户判断逻辑
- 渐进式自动化
- 用户信任积累

### 负面
- 阶段 1 用户操作多
- 阶段 2/3 实现复杂

### 风险
- 自动判断错误
- 缓解：阶段 2 时仍允许用户覆盖
