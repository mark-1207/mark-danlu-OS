# D-004: 8 项追问产出格式

## 状态
已敲定 | 2026-06-15

## 背景
苏格拉底追问的产出格式决定了追问的"价值"和后续步骤的输入。需要决定：
- 自由格式 vs 固定模板
- 几项产出
- 覆盖哪些维度

## 备选方案

### A：自由格式
- 用户说什么就什么
- 实现简单
- 价值低（没法结构化）

### B：固定模板（字段）
- 例：背景/挑战/解决方案/效果
- 4-6 个字段
- 偏"产品分析"风格

### C：8 项产出（炉选定）
- 命题浅层 / 底层逻辑 / 潜在诉求 / 风格建议
- 反共识点候选 / 思想框架候选 / 风险边界 / 可证伪性
- 覆盖"输入层 + 思考层 + 输出约束"

## 选定
**8 项产出**。

理由：
1. 4 项核心（直接服务蓝图设计）
2. 4 项辅助（反共识/框架候选/风险/可证伪性）
3. 每项都有"为什么是这一项"的逻辑
4. 总结是结果不是模板（用户明确"总结是追问之后的结果"）

## 后果

### 正面
- 8 项直接服务后续 5 步
- 字段可结构化 → pydantic schema
- 追问有明确"够"的标志

### 负面
- 8 项比较多，要给用户看摘要
- 实现：1 次 LLM 调用综合 8 项

### 风险
- 8 项产出失败 → 影响后续
- 缓解：失败时降级到 4 项核心

## 实施

详见 [99 P11](99-LESSONS-LEARNED.md) + [02-ARCHITECTURE 第 2.2 节](02-ARCHITECTURE.md)

```python
class RefinedProposition(BaseModel):
    surface: str                       # 1. 命题浅层
    underlying: str                    # 2. 底层逻辑
    audience: str                      # 3. 潜在诉求
    style_recommendation: StyleRecommendation  # 4. 风格建议
    contrarian_candidates: List[ContrarianPoint]  # 5. 反共识候选
    framework_candidates: List[FrameworkCandidate]  # 6. 思想框架候选
    risks: List[str]                   # 7. 风险点/边界
    falsifiability: str                # 8. 可证伪性
```
