# D-006: 配置化（YAML / 飞书表格）

## 状态
已敲定 | 2026-06-15

## 背景
思想模型/框架/必避免列表 怎么存，怎么管理。需要决定：
- 硬编码 vs 配置化
- 本地 vs 云端
- 何时升级

## 备选方案

### A：硬编码
- 模型定义在 Python 代码里
- 实现简单
- ❌ 改一个模型要改代码

### B：本地 YAML（v1）
- 配置文件
- 改动不影响代码
- ✅ 简单可控

### C：飞书多维表格（v2+）
- 可视化编辑
- 跨设备同步
- ✅ 灵活

### D：数据库
- 复杂
- v1 不需要

## 选定
**v1：本地 YAML** + **v2+：飞书多维表格**。

理由：
1. v1 单用户，本地 YAML 足够
2. v2+ 飞书表格提供可视化 + 同步
3. 不绑死实现，可演进

## 后果

### 正面
- 思想模型/框架/必避免列表可热更新
- 不需重启
- 多人协作（v2+）

### 负面
- 配置 schema 需要校验
- 飞书 API 限流

### 风险
- 飞书 API 失败
- 缓解：本地缓存 + 重试
