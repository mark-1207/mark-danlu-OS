# LLM 调用优化（v1.1.1, 2026-06-12）

> 30 → ~12 calls per run，~60% 降幅。不牺牲质量。

## 总览：4 批优化

| 批次 | 风险 | 节省 calls | 改动 |
|------|------|-----------|------|
| **B1 P0** | 零/低 | 2 | 删 prenarrative + expansion |
| **B2 P1** | 低 | 3 | Twin learn+filter 合并 + Prism 4→2 组 |
| **B3 P2** | 中（含防御）| 13 | CCOS L4+L8 合并 + Logic 12→1 |
| **总计** | — | **~18** | — |

## B1 P0 — 删除/消除无意义调用

### F. 删 prenarrative 调用
- `content_generator.py:2155` 之前调 `generate_prenarrative()`
- 标准 pipeline 不显示 prenarrative，直接进 full_draft
- **省 1 call**
- **风险：零**（删除的是未使用的调用）

### G. 删 expansion 靠 max_tokens
- `content_generator.py:2169` 之前 < 2500 字触发 `expansion_prompt` 二次调用
- writing-cn scene 已有 16384 max_tokens，prompt 加 "必须达到 2500-3000字" 约束
- **省 0-1 call**
- **风险：低**（一次输出更长，质量不变）

## B2 P1 — 低风险合并

### D. Twin learn + filter 合并
- `cognitive_crack.learn_and_filter_combined(candidates)` — 1 次 LLM 同时产出：
  - thinking_pattern / dimension_weights / style_keywords
  - selected_topics / rejected_topics
- `phases/twin.py` 改用新函数
- **省 1 call**
- **风险：低**（同 input，正交输出）

### A-modified. Prism 4 维 → 2 组
- `prism_engine.DIMENSION_GROUPS`:
  - challenges: reversal + contrarian
  - grounded: benefit_anchor + micro_scene
- `generate_dimension_group(thesis, [dim1, dim2])` — 1 次 LLM 返回 6 标题
- **省 2 calls**
- **风险：低**（语义分组焦点更集中）
- **fallback**：失败/不足时退化到 2 次单维

## B3 P2 — 中风险合并（带容错）

### C. CCOS Layer 4 + Layer 8 合并
- `cognitive_outline.generate_module_flow_and_energy(topic, structure, authorial_identity, platform)` — 1 次 LLM 同时产出 module_flow + narrative_energy
- L3 (推进方式) 必须先跑（输入 L4），所以 L3 单独保留
- **省 2 calls**（per both 平台）
- **风险：中低**（容错：任意一段缺失就 fallback 对应单段函数）

### E. Logic 12→1 合并（带 3 层防御）
- `logic_pressure._audit_batch_single(titles, retry)` — 1 次 LLM 返回数组
- `logic_pressure.audit_batch(titles)` — 3 层防御：

```python
# 第 1 层：1 次 LLM 调用
verdicts = _audit_batch_single(titles, retry=False)

# 第 2 层：长度不匹配 → retry 1 次
if len(verdicts) != n:
    verdicts = _audit_batch_single(titles, retry=True)

# 第 3 层：仍不匹配 → 对缺失项逐个 audit_title
if len(verdicts) != n:
    for t in titles:
        if t not in covered_titles:
            single = audit_title(t)  # 兜底
            verdicts.append({"title": t, **single})
```

- **省 11 calls**（最坏情况退化 ~3-4 calls）
- **风险：中**（12 项注意力分散，3 层防御保证最坏不退化）
- **测试**：`tests/test_logic_pressure_batch.py` 5 个测试覆盖防御层

## 不做的优化（明确说明）

| 优化 | 不做原因 |
|------|----------|
| 合并 Layer 2 三个 parallel → 1 | 已是 ThreadPoolExecutor 并行，墙钟一样（max 而非 sum）。仅省 quota 不省时间 |
| 4 维 prism → 1 维 | prompt 4× 长度（4000→16000 token），LLM 注意力分散，**质量风险不可忽略** |
| 合并 gateway questions + directions | 触发条件不同（clarify 才问，pass 只给方向），合并会浪费 |
