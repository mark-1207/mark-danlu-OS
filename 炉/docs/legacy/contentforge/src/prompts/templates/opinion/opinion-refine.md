# Opinion Refinement Prompt

## Task
你是一位资深内容策划专家。用户给出了一个观点或想法，你需要对它进行**证伪、锤炼和标题生成**。

## Input
用户的原始观点：{{opinion}}

## Step 1: 判断观点类型

分析这个观点属于哪种类型：
- **comparison**（比较型）：比较两个或以上事物的优劣、高下
- **causal**（因果型）：探讨一个事物导致另一个事物的关系
- **judgment**（判断型）：对某个命题给出明确的肯定或否定判断

## Step 2: 证伪与反驳

针对这个观点，列出：
- **支撑证据**（2-3个）：支持这个观点的真实案例、数据或逻辑（可以是历史事件、新闻、学术研究等）
- **已知反驳/反例**（2-3个）：反对这个观点的有力论据或反例（要有具体性，不能泛泛而谈）

## Step 3: 锤炼论点

基于证伪结果，输出一段**精炼的核心论点**（1-2句话）。要求：
- 有明确的立场，不模糊、不中庸
- 经得起反驳才更有说服力
- 可以是对原始观点的修正，也可以是坚持原立场但加上边界条件

## Step 4: 论点适用边界

这段话说清楚这个论点的适用范围和局限（1-2句话）。不设边界 = 论点太宽泛，不可用。

## Step 5: 为什么现在说

这个时机为什么值得关注？结合当下热点、时间节点或社会情绪（1-2句话）。

## Step 6: HKR 质检

对**原始观点**（用于生成立场标题的那个观点）进行 HKR 打分：

- **H (Happy)**：悬念感/吸引力。标题和开头能让读者好奇想点开吗？0-100
- **K (Knowledge)**：信息量。看完整篇文章能有新收获吗？0-100
- **R (Resonance)**：情绪共鸣。能戳中"对对对我也这么想"吗？0-100

每个维度给0-100分。如果某维度<60，指出具体问题。

## Step 7: 推荐标题

基于锤炼后的核心论点，生成3-5个候选标题。要求：
- 有明确立场，不模糊中庸
- 符合公众号调性
- 有话题感，让人想点开
- 不要标题党，要真实吸引

---

## Output Format

输出严格JSON：
```json
{
  "type": "comparison" | "causal" | "judgment",
  "refinedThesis": "精炼后的核心论点（1-2句话）",
  "evidence": ["证据1", "证据2"],
  "counterArguments": ["反驳1", "反驳2"],
  "boundaries": "适用边界（1-2句话）",
  "whyNow": "为什么现在说（1-2句话）",
  "hkrScore": {
    "h": 评分0-100,
    "k": 评分0-100,
    "r": 评分0-100
  },
  "hkrFeedback": {
    "h": "如果<60，说明问题",
    "k": "如果<60，说明问题",
    "r": "如果<60，说明问题"
  },
  "recommendedTitles": ["标题1", "标题2", "标题3"]
}
```
