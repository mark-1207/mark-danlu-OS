# 09 — v2 规划（Roadmap v2）

> 版本：v1.0 草稿 | 状态：方案已对齐
> 不含"多人使用"相关（v3 长期）

## 1. 总则

### 1.1 v2 是什么

v2 = 在 v1 基础上的**单用户延伸**：
- 配置管理升级（飞书表格）
- 学习机制升级（阶段 2-3）
- 能力扩展（Embedding/高级策略/复盘）

### 1.2 v2 不做什么

- ❌ 多用户
- ❌ 账号体系
- ❌ 权限管理
- ❌ 多租户
- ❌ Web 化
- ❌ 计费/订阅

（以上放 v3 长期，见 `10-LONG-TERM-PLAN.md`）

### 1.3 v2 优先级

P0 = 必做 / P1 = 应该做 / P2 = 评估

## 3. P0：阶段 2-3 学习机制

### 3.1 目标
- 阶段 2（30-100 样本）：系统提示"可收尾"，用户决定
- 阶段 3（100+ 样本）：自动判断，不再问

### 3.2 实现

- `socratic/learning.py` 扩展
- 样本存储：本地 `config/socratic_samples.jsonl`
- 特征工程：追问轮数、已挖字段、用户最后一句话
- 模型：轻量分类（logistic / decision tree）
- 训练：每 10 个样本重训一次

### 3.3 验收
- 阶段 2 命中率 ≥ 80%（用户说"判断对了"的比例）
- 阶段 3 命中率 ≥ 90%

### 3.4 工期
1 周

## 4. P0：Embedding 语义匹配

### 4.1 目标
- 跨命题主题聚合（"杠杆者"标签下的所有案例/金句/洞察）
- 智能召回：写"AI 牛马"时召回"杠杆者"相关历史素材
- 命题相似度检测：避免重复命题

### 4.2 实现

- Embedding Provider：智谱（zhipu embedding-3, 2048 维）→ openai → tavily → google
- 索引：`obsidian/index/`（本地向量索引）
- 召回：cosine similarity 阈值 ≥ 0.7
- 触发：Step 2 追问完成后 / Step 3 蓝图设计前

### 4.3 工期
1-1.5 周

## 5. P1：高级策略模式

### 5.1 目标
- `iterate`（迭代）：用 A 得 B，再用 B 修 A
- `recursive`（递归）：模型套自己

### 5.2 典型场景

**iterate**：
- 第一性原理拆解 → 反向检验 → 调整拆解
- 蓝图设计 → 用户反馈 → 重设计

**recursive**：
- 用"第一性原理"拆"什么是第一性原理"
- 用"反共识"找"反共识的反共识"

### 5.3 实现

- `thinking_models/model_runner.py` 扩展
- 添加 `_run_iterate` / `_run_recursive` 方法
- 迭代上限：3 次（防无限循环）
- 递归上限：2 层（防栈溢出）

### 5.4 工期
1-1.5 周

---

## 6. P1：复盘/雷达/周报生成

### 6.1 复盘
- 每周/月跑过的 run 自动复盘
- 评分趋势 / 必避免命中趋势 / 时间趋势
- 输出：复盘报告 Markdown

### 6.2 雷达
- 主动推送可能的命题（基于历史 + 热点）
- 每周 1-3 个候选命题
- 用户选择后启动 run

### 6.3 周报生成
- 每周 1 篇"内容创作周报"
- 包含：写了什么 / 评分趋势 / 沉淀了什么
- 可选自动发到飞书/邮箱

### 6.4 工期
2-3 周（可拆分）

## 7. P1：自定义模型

### 7.1 目标
- 用户自定义思想模型（CLI `炉 model add`）
- 用户自定义框架（CLI `炉 framework add`）
- 自定义后立即生效（不需重启）

### 7.2 实现

- CLI 子命令：`炉 model add` / `炉 framework add`
- TUI 表单：填定义/使用场景/正例/反例
- 存储：本地 YAML（v1）/ 飞书表格（v2）
- 校验：避免重复 ID / 字段缺失

### 7.3 工期
1 周
