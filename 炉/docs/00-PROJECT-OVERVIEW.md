# 炉 — 项目总览（Project Overview）

> 版本：v1.0 草稿 | 状态：文档完整，v1 待启动

## 一、一句话定位

**炉** = 你的内容风格化创作引擎：从一个命题到一篇有"你"的声音的文章，7 步闭环跑通。

## 二、核心问题（要解决什么）

| 问题 | 表现 |
|------|------|
| 有了选题但写不出来 | 命题有了，落笔就卡 |
| AI 味儿太浓 | 套话/节奏呆板/无个人视角/比喻陈旧/无反共识/数据堆砌 |
| 个人风格难沉淀 | 这次写得好，下次又变 AI 味儿 |
| 质量不稳定 | 时好时坏，依赖灵感和状态 |
| 认知不清 | 命题模糊就开始写，写完才发现在论证"错的东西" |

## 三、目标用户

| 阶段 | 范围 |
|------|------|
| **v1（当前）** | 单用户：mark（独立开发者） |
| v2+ | 仍为单用户延伸（飞书表格管理配置 + 跨设备同步） |
| **v3 长期** | 开放给其他人使用（账号体系/权限/多租户/Web 化/计费） |

## 四、核心能力

1. **苏格拉底追问**：3-5 轮固定模板 + 动态调整，输出 8 项认知澄清
2. **思想模型框架**：4 框架（问题解构/决策分析/系统思考/创新突破）+ 12 模型卡片
3. **7 步循环**：命题输入 → 追问澄清 → 蓝图设计 → 段位选择 → 草稿生成 → 打磨质检 → 沉淀回写
4. **8 段弹性结构**：核心 5 必选 + 可选 8 按内容类型推荐 2-4
5. **L5 三子项评分**：观点锐度 / 思想模型应用 / 事实准确性
6. **飞书工作流**：灵感池（原始洞察录入）+ 反馈数据（实际表现回写）
7. **Obsidian 素材库**：跨命题复用的案例/数据/金句/思想模型
8. **爆款二创**：从已有爆款重写（ViralGenome 拆解 + 差异化重写）

## 五、边界（什么/不是什么）

### 做
- 命题→创作的全流程
- 认知澄清（追问）
- 蓝图设计（CCOS 14 项 → 8 段渲染）
- 多模型思想框架
- 风格画像沉淀
- 飞书多维表格（v2 阶段）
- Obsidian 本地素材库
- 爆款二创

### 不做（v1 砍掉）
- ~~发布到平台~~（公众号/小红书/抖音自动发）
- ~~自动搬运~~（爬虫+改写）
- ~~多用户/账号体系~~（v3 长期）
- ~~Web 化/服务化~~（v3 长期）
- ~~计费/订阅~~（v3 长期）

## 六、源项目关系

| 项目 | 类型 | 状态 |
|------|------|------|
| **ContentForge** | TypeScript | 源项目继续独立迭代，代码快照在 `docs/legacy/contentforge/` |
| **PRISM-OS** | Python | 源项目继续独立迭代，代码快照在 `docs/legacy/prismos/` |
| **炉** | Python | 独立仓库，独立迭代，从源项目代码快照作为基线 |

**物理隔离原则**：
- 源项目 + 新项目 = 不同 git 仓库
- 炉 的代码改动**不反向同步**到源项目
- 源项目的新能力**不自动同步**到炉
- 双向能力借鉴 = 手动复制 + 决策

## 八、技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 语言 | Python 3.11+ | 用户偏好单边；PRISM 用 Python |
| 飞书 | lark-cli subprocess | 避免 SDK 依赖，灵活 |
| 配置 | 本地 YAML（v1）/ 飞书表格（v2+） | 分阶段推进 |
| LLM | 待定（v1 评估 openai/kimi/anthropic） | 沿用 ContentForge 链 |
| 持久化 | 本地文件（v1）/ 数据库（v2+） | 分阶段 |
| 部署 | CLI（v1）/ 服务化（v3 长期） | 分阶段 |

## 九、验收标准

### 硬指标
- **8 维度评分**（6 维 + L5 三项）每项 ≥ **7.5**

### 软指标（最高标准）
- **你亲自读完**判断"这就是我的声音" — **人工验收为最高标准**

### 时间
- 涉及人的参与（修改内容等）≤ 1 小时
- 系统时间要快，优先"跑通 + 质量好 + 速度不慢"

## 十二、术语表

| 术语 | 含义 |
|------|------|
| 命题 | 用户输入的初始创作主题（如"AI 牛马陷阱"） |
| 追问 | 苏格拉底式提问，澄清命题的 8 个维度 |
| 蓝图 | CCOS 14 项 + 8 段渲染的结构化大纲 |
| 框架 | 思想模型框架 = 多个模型组合的决策策略包 |
| 模型 | 单个思维工具（杠杆者/第一性原理/...） |
| 锚点 | 蓝图中的具体素材（案例/反共识/数据/洞察/金句） |
| 段位 | 一篇文章的段落角色（钩子/反共识/案例/...） |
| 风格画像 | 用户的语言指纹 + 必避免列表 + 偏好沉淀 |
| L5 | 评分维度的第 5 层（观点锐度/思想模型应用/事实准确性） |
| 沉淀 | 写完一篇后，把产物回写到风格画像 + Obsidian 素材库 |
