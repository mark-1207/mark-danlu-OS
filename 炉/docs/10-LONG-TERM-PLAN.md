# 10 — 长期规划（Long Term Plan, v3+）

> 版本：v1.0 草稿 | 状态：方案已对齐
> 包含"多人使用"等 v3+ 才会做的事

## 1. 总则

### 1.1 长期是什么

长期 = **产品化**：
- 从单用户工具 → 多用户产品
- 从 CLI → Web/服务化
- 从免费 → 计费/订阅

### 1.2 长期不做（v1/v2 已经决定）

- ❌ 自动发布到平台
- ❌ 自动搬运/爬虫改写
- ❌ 内容农场模式

### 1.3 长期核心定位

> "你的内容风格化创作引擎" → 走向认知复利 + 个人 IP

核心价值不变（让用户写出"自己的声音"），但承载方式从"个人工具"变成"产品"。

## 2. v3.x 路线图

### 2.1 v3.0：开放 + 账号体系

**目标**：让其他人能用 炉

**范围**：
- Web 前端（Next.js）
- 后端 API（FastAPI）
- 账号体系（邮箱/手机号 + 密码）
- 飞书/微信 OAuth 登录
- 多租户数据隔离
- 基础文档站

**工期**：6-8 周

**风险**：
- 部署成本（云服务费用）
- 安全（账号/数据/审计）
- 客服/运营

### 2.2 v3.1：服务化

**目标**：从单机 CLI → 后端服务

**范围**：
- Celery + Redis 任务队列
- PostgreSQL 替代文件存储
- S3 / 阿里云 OSS 替代本地文件
- Docker + Kubernetes 部署
- API 限流 + 监控
- 报警（Prometheus + Grafana）

**工期**：4-6 周

### 2.3 v3.2：计费/订阅

**目标**：商业化

**范围**：
- 订阅制（月费/年费）
- 免费额度（每月 10 篇）
- 按 LLM 调用计费
- 支付宝/微信支付
- 发票系统

**工期**：3-4 周

### 2.4 v3.3：思想模型库社区化

**目标**：社区共建

**范围**：
- 公共仓库
- 评分/评论
- 一键导入
- 模型版本管理
- 必避免列表社区维护

**工期**：3-4 周

### 2.5 v3.4：内容创作生态

**目标**：扩展使用场景

**范围**：
- 复盘（已做）
- 雷达（已做）
- 周报（已做）
- **内容日历**：跨周/月内容规划
- **AI 协作**：多人共同创作一篇文章
- **风格借鉴**：用 X 的风格写
- **历史回溯**：所有 run 永久保存

**工期**：4-6 周

## 3. 多用户产品架构

### 3.1 总体架构

```
┌─────────────────────────────────────────────┐
│  Browser (Next.js)                          │
│  - 登录/注册                                │
│  - run 列表/详情                            │
│  - 风格画像编辑                             │
│  - 飞书配置同步                             │
└─────────────────────────────────────────────┘
              ↓ HTTPS
┌─────────────────────────────────────────────┐
│  API Gateway (FastAPI)                      │
│  - 认证 (JWT)                               │
│  - 限流                                     │
│  - 审计日志                                 │
└─────────────────────────────────────────────┘
              ↓
┌──────────────┬──────────────┬──────────────┐
│  Web 服务     │  Celery Worker│  Scheduler   │
│  (FastAPI)    │  (LLM 调用)  │  (雷达/周报) │
└──────────────┴──────────────┴──────────────┘
              ↓
┌─────────────────────────────────────────────┐
│  PostgreSQL / Redis / S3 / LLM              │
└─────────────────────────────────────────────┘
```

### 3.2 数据模型（多用户版）

```sql
-- 用户
users (
  id, email, password_hash, created_at
)

-- 用户的 run
runs (
  id, user_id, proposition, state, context_json, created_at
)

-- 用户的风格画像
style_profiles (
  user_id, fingerprint, forbidden_list, version
)

-- 用户的素材库
materials (
  id, user_id, type, content, tags, embedding, created_at
)

-- 订阅
subscriptions (
  user_id, plan, expires_at, status
)

-- 计费记录
billing (
  id, user_id, amount, type, created_at
)
```

### 3.3 安全

- **认证**：JWT + Refresh Token
- **授权**：RBAC
- **数据隔离**：每用户独立 namespace
- **API 限流**：每用户/每 IP
- **审计日志**：所有操作留痕
- **加密**：传输 HTTPS + 存储加密
- **备份**：每日备份 + 异地容灾

## 4. Web 化产品设计

### 4.1 信息架构

```
首页（登录前）
  - 价值主张
  - 案例
  - 定价
  - 登录/注册

工作台（登录后）
  - 我的 Run
  - 风格画像
  - 素材库
  - 飞书配置
  - 订阅/计费

Run 详情
  - 7 步流程
  - 实时评分
  - 编辑/导出

风格画像
  - 语言指纹
  - 必避免列表
  - 历史

素材库
  - 案例
  - 金句
  - 洞察
  - 思想模型

设置
  - LLM Provider
  - 飞书绑定
  - 订阅
```

### 4.2 关键页面

- **首页**：价值主张 + CTA（"开始创作"）
- **工作台**：run 列表 + 快速启动
- **7 步流程**：每步有引导、有撤销
- **风格画像编辑**：可视化 + 实时预览
- **素材库搜索**：全文 + 标签 + Embedding

### 4.3 用户体验

- **响应时间**：< 2s（页面）
- **LLM 调用**：进度条 + 可取消
- **多设备**：响应式 + 数据同步
- **离线**：v3.4+ 评估 PWA

## 7. 长期风险

### 7.1 技术风险

| 风险 | 缓解 |
|------|------|
| LLM 成本失控 | 限流 + 缓存 + 模型分级 |
| 数据丢失 | 每日备份 + 异地容灾 |
| 安全漏洞 | 渗透测试 + 漏洞赏金 |

### 7.2 产品风险

| 风险 | 缓解 |
|------|------|
| 用户增长缓慢 | 内容运营 + 社区 |
| 留存低 | 必避免列表沉淀 + 风格画像迁移成本 |
| 付费转化低 | 免费试用 + 教育市场 |

### 7.3 商业风险

| 风险 | 缓解 |
|------|------|
| 政策风险（AI 内容） | 透明 + 用户责任 |
| 竞品 | 差异化（思想模型 + 风格画像） |
| LLM Provider 涨价 | 多 Provider 链 + 自托管选项 |

## 8. 长期组织

### 8.1 团队

- **早期**（v3.0）：mark 单人 + 1-2 个外包
- **中期**（v3.2）：+ 全职工程师 1-2 人
- **后期**（v3.4+）：+ 产品 / 运营 / 客服

### 8.2 现金流

- **v1/v2**：自给（无收入）
- **v3.0**：开始有付费用户
- **v3.2+**：订阅收入增长
- **v3.4+**：盈亏平衡 / 盈利

---

## 9. 不做的事（明确砍）

- ❌ 不做"内容农场"（批量生成低质内容）
- ❌ 不做"伪原创"（爬虫+改写）
- ❌ 不做"代写"（替用户写无风格的内容）
- ❌ 不做"AI 绘画"等扩展（聚焦写作）
- ❌ 不做"短视频"等其他媒体（聚焦图文）
