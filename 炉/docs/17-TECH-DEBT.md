## 🆕 T-001: v1 阶段不接 CI

**类型**：基础设施

**背景**：
- v1 阶段手动跑测试
- v2 阶段才上 GitHub Actions

**解决方案**：
- 写 `.github/workflows/test.yml`（v2 阶段）

**优先级**：P2（v2 才做）

---

## 🆕 T-002: 没有命令行自动补全

**类型**：UX

**背景**：
- CLI 命令多，用户记不住
- 可加 `argcomplete` 或 `click` 的 shell completion

**优先级**：P2

---

## 🆕 T-003: 没有 run 列表的搜索功能

**类型**：功能

**背景**：
- `炉 history` 只列 run_id 和 proposition
- 没法搜索

**优先级**：P2

---

## 🆕 T-004: LLM 调用串行，未做并发

**类型**：性能

**背景**：
- 8 维度评分串行 8 次 LLM 调用
- 可并发加速

**解决方案**：
- 用 `asyncio.gather` 并发
- 注意 LLM Provider 限流

**优先级**：P1

---

## 🆕 T-005: 配置文件无 schema 校验

**类型**：健壮性

**背景**：
- `config/style_profile.yaml` 没有 schema 校验
- 用户改坏会导致启动失败

**解决方案**：
- 用 pydantic 加载
- 失败时报错 + 友好提示

**优先级**：P1

---

## 🆕 T-006: 必避免列表是字符串数组，不是结构化

**类型**：架构

**背景**：
- 当前 `forbidden_list: List[str]`
- 但每个禁词应该带：词/类别/严重度/替换建议

**解决方案**：
- 改为 `ForbiddenTerm` 对象
- 向后兼容：字符串自动转对象

**优先级**：P1

---

## 🆕 T-007: 没有 run 之间的 diff 对比

**类型**：功能

**背景**：
- 用户想看两篇 run 的差异
- 当前没法对比

**优先级**：P2

---

## 🆕 T-008: 没接 LLM 成本预算控制

**类型**：成本

**背景**：
- 跑一篇可能花几毛钱到几块钱
- 累计可能失控

**解决方案**：
- 单篇预算（默认 ¥1）
- 超出报警 + 暂停

**优先级**：P1

---

## 🆕 T-009: 没接 LLM 调用缓存

**类型**：性能/成本

**背景**：
- 相同 prompt 多次调用 → 浪费钱
- 可加缓存层

**解决方案**：
- 用 prompt hash 作为 key
- LRU 缓存（默认 1000 条）

**优先级**：P1

---

## 🆕 T-010: 没接 LLM 调用 trace

**类型**：可观测性

**背景**：
- LLM 调用没有 trace
- 出问题难定位

**解决方案**：
- 用 OpenTelemetry / LangSmith
- 记录每次调用的 prompt/response/时间

**优先级**：P2
