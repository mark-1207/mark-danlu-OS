# 16 — 发布清单（Release Checklist）

> 发版前必走流程。v1 简化：手动跑。
> 状态：⬜ 未做 / ✅ 已做 / ❌ 跳过（说明原因）

---

## 一、自动化检查

### 1.1 测试

- [ ] 跑全量单元测试：`pytest`
- [ ] 跑全量集成测试
- [ ] 跑端到端（mock LLM）
- [ ] 跑端到端（真实 LLM，慢测试）
- [ ] 覆盖率 ≥ 80%（核心模块 100%）

### 1.2 类型检查

- [ ] `mypy src/ --strict`

### 1.3 Lint

- [ ] `ruff check src/`
- [ ] `black --check src/`
- [ ] `isort --check src/`

### 1.4 文档

- [ ] 19 个文档无 TODO 占位
- [ ] CHANGELOG 更新（v1 简化：可不做）
- [ ] 决策日志更新（[12-DECISION-LOG](12-DECISION-LOG.md)）
- [ ] 进度日志更新（[11-PROGRESS](11-PROGRESS.md)）

---

## 二、人工检查

### 2.1 功能验证

- [ ] 7 步全跑：标准命题（如"AI 牛马陷阱"）
- [ ] 8 维度评分 ≥ 7.5
- [ ] 必避免列表命中 ≤ 2/篇
- [ ] 续跑：中断后能续
- [ ] 风格画像更新：跑完后画像有变化

### 2.2 性能验证

- [ ] 7 步全跑时间 ≤ 10 分钟
- [ ] 单步平均 ≤ 90 秒
- [ ] LLM 调用/篇 ≤ 15 次
- [ ] 单次 LLM 调用 ≤ 30 秒

### 2.3 安全验证

- [ ] .env 不在 git
- [ ] 日志不输出 API key
- [ ] 用户输入有验证
- [ ] 文件路径防 path traversal

### 2.4 用户体验

- [ ] mark 亲自读完文章
- [ ] 必避免列表符合预期
- [ ] 风格画像反映"我"的声音
- [ ] 命题 → 成文流畅

### 2.5 异常路径

- [ ] LLM 失败 → 重试 → fallback
- [ ] 持久化失败 → 不丢数据
- [ ] 必避免列表漂移 → 报警
- [ ] 续跑（断电/中断）

---

## 三、发版动作

### 3.1 v1 阶段（手动）

- [ ] commit 所有变更
- [ ] 跑发版清单（本文件）
- [ ] 标记 milestone（[13-MILESTONES](13-MILESTONES.md)）
- [ ] 备份 runs / config / obsidian
- [ ] 更新 PROGRESS.md
- [ ] 用户验收签字

### 3.2 v2 阶段（自动化）

- [ ] tag version
- [ ] GitHub release
- [ ] 部署到服务
- [ ] 通知用户
- [ ] 监控告警

---

## 四、回滚

### 4.1 触发条件

- 8 维度评分 < 7.5
- 必避免列表命中率 > 5/篇
- 7 步全跑时间 > 20 分钟
- 用户体验严重下降

### 4.2 回滚动作

- [ ] 回滚代码（git revert 或 git reset）
- [ ] 恢复备份
- [ ] 通知用户
- [ ] 写 POST-MORTEM（追加到 [15-ISSUES](15-ISSUES.md)）

---

## 五、发版后

- [ ] 观察 1 周用户使用情况
- [ ] 收集反馈
- [ ] 写经验教训（追加到 [99-LESSONS-LEARNED](99-LESSONS-LEARNED.md)）
- [ ] 规划下个版本

---

## 维护规范

- 每次发版前打印本文档
- 强制执行清单
- 跳过项必须有原因
- 完成后归档到 `docs/releases/<version>/`
