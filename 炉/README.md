## 关键原则

1. **物理隔离**：源项目 + 炉 独立迭代，不双向同步
2. **配置化优先**：思想模型/框架/必避免列表走 YAML（v1）/ 飞书表格（v2+）
3. **不硬编码**：每个模块可配置
4. **文档先于代码**：PRD/架构/规范/计划/测试 优先于实施
5. **TDD + 端到端验证**：每项 commit 必跑全量 test

## 验收标准

- **8 维度评分**（6 维 + L5 三项）每项 ≥ 7.5
- **人工验收**：mark 亲自读完判断"这就是我的声音"
- **时间**：人参与 ≤ 1h，系统时间要快
