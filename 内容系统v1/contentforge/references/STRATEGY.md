# ContentForge 战略讨论记录

> 记录与 mark 的深度讨论，时间 2026-04-21

## 一、skill 封装的价值重估

**原始认知**：skill 只是降低"说人话"的门槛，体验层简化，无能力增强。

**修正后的认知**：skill 的价值分两层：
- **体验层**：自然语言入口、统一汇报格式
- **能力层**：碎片库持久学习、多步骤 pipeline 状态传递、病毒基因提取 → 差异化方向 → 双审查的结构化质量门控、素材搜索、平台适配、成本控制、断点续跑

**核心结论**：碎片库的持久学习（引用+decay）不是 prompt 能替代的，这是真正的存量积累。

## 二、ContentForge 应具备但被忽略的能力（8点）

### 1. 内容迭代与修订
- 增量修订（只改 hook，不重跑全流程）
- 多轮对话式修订（一来一回打磨内容）
- 修订历史与版本管理

### 2. 用户写作风格建模
- 风格画像（句子长度、词汇习惯、情绪调性）
- 差异化定位分析（领域覆盖建议）
- 风格一致性检测（新内容与历史风格偏差预警）

### 3. 竞争分析与选题
- 竞品爆款分析（搜索同主题爆款，分析为什么火）
- 选题推荐（基于领域积累 + 热度分析）
- 热度归因（追踪热点来源）

### 4. 合规与风控
- 原创性风险检测（版权阈值对标）
- 平台规则适配（敏感词检测）
- 敏感内容分类（政治、医疗、金融等）

### 5. 发布与分发
- 草稿自动发布（推送到平台草稿箱）
- 发布时间建议（基于受众活跃时间）
- 发布后数据追踪（阅读/点赞/转发回写）
- 多平台差异化发布时间

### 6. 团队协作
- 评论管理与 AI 回复生成
- 多账号管理（个人号/机构号切换）
- 审稿流程（提交→审稿意见→修改→确认发布）

### 7. 内容资产管理
- 内容资产台账（标题/平台/关键词/发布时间/效果数据）
- 标签体系（领域/情绪/形式/阶段）
- 内容重组复用（跨平台碎片改编）

### 8. 可观测性与学习闭环
- 生成效果追踪（发布后数据回写，校准评分模型）
- A/B 测试（双角度生成，收集数据学习最优方向）

## 三、skill 架构讨论

### 问题

这 8 点 + 当前 skill 的能力，如果全塞进一个 skill，会太重、难以维护、难以演进。

### 结论

应该拆成多个专注的 skill，形成**skill 矩阵**：

| Skill | 触发 | 职责 |
|-------|------|------|
| `/contentforge` | 通用内容生成 | 核心生成 pipeline，现状不变 |
| `/contentforge-revision` | "改一改"、"再优化一下" | 增量修订 + 多轮对话打磨 |
| `/contentforge-style` | "分析我的写作风格" | 风格画像 + 一致性检测 |
| `/contentforge-topic` | "有什么可以写的" | 竞品分析 + 选题推荐 |
| `/contentforge-compliance` | "帮我检测下这篇有没有问题" | 原创性 + 敏感词 + 平台规则检测 |
| `/contentforge-publish` | "发到公众号" | 发布到草稿箱 + 时间建议 + 数据追踪 |
| `/contentforge-asset` | "看看我写过什么" | 内容台账 + 标签体系 + 跨平台复用 |
| `/contentforge-team` | "让小李审稿" | 审稿流程 + 多账号 + 评论管理 |

### 设计原则

1. **单一职责**：每个 skill 只管一件事，不要交叉
2. **共享数据层**：`output/corpus/`、`output/fragment-library.json` 等碎片库所有 skill 共享
3. **按需加载**：用户不需要的 skill 不会拖累主流程
4. **自然触发**：`description` 写清楚触发词，AI 自动路由到对的 skill

### 优先级建议

| 优先级 | Skill | 理由 |
|--------|-------|------|
| P1 | `/contentforge-revision` | 解决"生成不满意怎么办"的核心痛点 |
| P1 | `/contentforge-style` | 让生成的内容开始"像这个人"，壁垒建立 |
| P2 | `/contentforge-topic` | 让内容有市场竞争意识，不闭门造车 |
| P2 | `/contentforge-compliance` | 避免医疗/金融/政治内容发布风险 |
| P3 | 其余 | 锦上添花 |

## 四、当前 skill 现状与待做

**已实现**：
- `/contentforge` — 自然语言统一入口（原创/二创/平台自动判断）
- `skill` CLI 命令 — 底层路由

**待规划（按优先级）**：
- P1: revision（迭代修订）
- P1: style（风格画像）
- P2: topic（选题推荐）
- P2: compliance（合规风控）
- P3: publish / asset / team
