# 内容反馈系统 PRD（CCOS-F v1.0）

## 1. 系统愿景与背景

### 愿景
基于真实用户行为数据，持续优化内容认知系统——通过数据溯因，理解「用户为什么会产生某种反应」，最终形成内容创作的学习闭环。

### 背景
ContentForge 当前已具备完整的内容生成能力（竞品抓取 → AI 分析 → 碎片提取 → 多平台创作）。缺失的一环：**生成的内容发布后表现如何，下次创作如何基于真实反馈优化**。

当前数据流：
```
竞品 → 飞书表 → AI分析 → 碎片 → Obsidian → create → 内容
```
缺失：
```
发布后数据 → 反馈分析 → Insight → 下一轮create
```

### 设计原则
1. **人读优先，机器可读其次**：报告首先服务于人工阅读和决策，machine-readable 结构等 Phase 3 再做
2. **数据质量先于分析复杂度**：统计分析在小样本量下不可信，先建数据采集习惯，再逐步升级分析维度
3. **反馈必须可操作**：每个分析结论都要有对应的「下一步行动」，不能停留在数字本身
4. **单向依赖，不循环**：竞品表和反馈表各自独立，不做跨表强关联，只做标签级差距分析

## 2. 用户故事

### 用户 A（创作者）
> 我发布了一篇文章到公众号，7 天后在后台看到数据，手动填入飞书反馈表。14 天后再填一次。30 天后再填一次。

### 用户 B（系统使用者）
> 我运行 `learn --feedback-summary`，看到近期表现最好的内容类型是「对比型叙事结构 + 励志调性」。
> 运行 `learn --include-competitor`，看到竞品报告说我最弱的标签是「效率」，因为竞品用「对比型结构」写效率内容互动率 8%，我方只有 3%。
> 我决定下一批内容优先用「对比型结构」写效率话题。

### 用户 C（系统）
> 我运行 `learn --include-competitor`，报告生成后存入 Obsidian 竞品库。
> 竞品表已有 analyzed 记录自动补填叙事结构/情感调性/内容角度。

## 3. 核心模块设计

### 模块 A：反馈数据表

**定位**：我方内容的核心数据采集入口

**数据结构**（已在飞书创建）：

| 内容角度 | 文本 | 切入角度描述 |
| 叙事结构 | 单选 | 故事型/清单型/对比型/分析型/混合型 |
| 情感调性 | 单选 | 励志/冷静/温暖/犀利/幽默 |
| 发布日期 | 日期 | |
| 数据周期 | 单选 | 7日/14日/30日 |
| 阅读量 | 数字 | |
| 点赞数 | 数字 | |
| 评论数 | 数字 | |
| 转发数 | 数字 | |
| 完播率 | 数字 | 视频号/小红书用 |
| 收藏数 | 数字 | 小红书核心指标 |
| 数据备注 | 多行文本 | 手工备注 |
| 下次更新时间 | 日期 | 30日数据提醒 |
| **文章ID** | 文本 | **Phase 1-1 新增**：同一文章跨周期追踪 |

**设计说明**：
- 同一文章按周期追加新行（旧数据不覆盖），支持纵向趋势追踪
- 文章ID 字段由用户手动填写（格式：如 `wc-2026-05-01-001`）
- 互动率不存储，用公式在分析时计算（避免数据不一致）

### 模块 C：竞品分析报告

**定位**：发现竞品高表现规律，指导内容策略调整

**触发命令**：`learn --include-competitor`

**输出位置**：
- `output/corpus/竞品库/竞品分析报告-{YYYY-MM-DD}.md`
- 同步复制到 Obsidian `竞品库/`

**报告结构**（6 Section + 附）：

| Section | 核心问题 | 性质 |
|---------|----------|------|
| Section 0 | 读完记住的 1 件事是什么 | 必须 |
| Section 1 | 我在市场的位置 vs 竞品在哪 | 必须 |
| Section 2 | 竞品高表现的结构/调性/角度/标签规律 | 必须 |
| Section 3 | 我方内容差在哪（标签级差距） | 必须 |
| Section 4 | 这一期最值得记住什么（可行动的洞察） | 必须 |
| Section 5 | 下一步具体怎么做（优先级排序） | 必须 |
| 附 | 数据来源说明 | 辅助 |

**生成逻辑**：
1. 读取竞品表所有 analyzed/stored 记录
2. 按标签/叙事结构/情感调性分组，计算各组平均互动率
3. 读取反馈表，计算我方各维度平均互动率
4. 对比竞品 vs 我方各标签表现差异
5. LLM 辅助生成 Section 0（核心发现）和 Section 4（关键洞察）

**不生成的内容**：
- 不做机器可读的 insight JSON
- 不自动注入 create pipeline
- 不做认知标签分析（Phase 2）

---

### 模块 D：反馈分析 CLI

**定位**：发现我方内容高低因素，找到与竞品的标签级差距

**触发命令**：
- `learn --feedback-summary` — 输出我方分析报告
- `learn --feedback-compare` — 输出竞品 vs 我方差距分析

**输出**：
- Console 输出（人读格式）
- 无持久化文件（每次分析实时计算）

**分析维度**（`computeFeedbackStats`）：
- 按平台：各平台平均阅读/互动率
- 按叙事结构：各结构类型平均互动率
- 按情感调性：各调性平均互动率
- 按内容角度：各角度平均互动率
- 按标签：各标签平均互动率

**差距对比**（`compareWithCompetitor`）：
- 同一标签下，我方 vs 竞品的互动率对比
- 差距 >阈值（1%）标记为显著差异，给出行动建议

---

### 模块 E：时序聚合（Phase 1-4）

**定位**：追踪同一文章的爆发力和长尾力

**数据结构扩展**：
- 反馈表按「文章ID + 周期」聚合
- 每篇文章计算：`趋势 = 7日互动率 → 30日互动率的变化`
- 趋势分类：`rising`（持续上升）/ `declining`（持续下降）/ `stable`（先升后降）

**用途**：
- 识别高爆发力内容（7日高但30日跌 = 标题党）
- 识别高长尾力内容（7日低但30日升 = 搜索流量型）
- 为创作方向提供时间维度参考

## 4. 竞品分析报告内容详解

### Section 0：核心发现一句话

**目标**：一行字让人记住本期最重要结论，用于快速决策

**格式**：`[竞品/我方] [发现] → [行动建议]`

**示例**：
```
竞品 70% 的爆款文章使用"对比型叙事结构"开头，
且该结构的竞品平均互动率比清单型高 3.2%，
我方使用对比型结构的文章仅占 12%。
→ 建议：下一批职场话题内容优先采用对比型叙事。
```

---

### Section 2：高表现规律

**结构规律子节**：
```
高互动竞品内容中，叙事结构分布：
  故事型：38%（平均互动率 8.1%）
  对比型：27%（平均互动率 7.6%）
  清单型：20%（平均互动率 5.2%）
  分析型：15%（平均互动率 4.8%）

为什么对比型有效：
  - 认知落差引发好奇（"我以为A，原来是B"）
  - 适合职场/效率/财富等实用类话题
  - 与"冷静分析型"读者调性契合
```

**角度规律子节**：
```
高互动竞品切入角度分布：
  "普通人逆袭"型：35%（互动率 8.3%）
  "反常识颠覆"型：25%（互动率 7.5%）
  "可量化结果"型：22%（互动率 6.8%）
  "问题解决"型：18%（互动率 6.1%）
```

---

### Section 5：下一步行动

**格式**：
```
行动优先级 | 具体动作 | 目标话题 | 预期效果

P1 | 切换叙事结构为对比型 | 职场晋升话题首批5篇 | 预计互动率提升 2-3%
P2 | 调整标题策略，增加数字型标题 | 效率话题 | 预计打开率提升
P3 | 持续跟踪30日数据，暂不下结论 | 视频号内容 | 待验证
```

## 5. 关键设计决策

### 决策 1：反馈数据表 vs 竞品表分离

**选择**：两个独立表，不做跨表强关联

**理由**：
- 竞品表记录外部内容，反馈表记录我方内容，数据性质不同
- 两者通过标签做语义关联（同一标签下的内容可对比），不通过 ID 关联
- 避免状态流转混乱（竞品表状态：pending→analyzed→stored，反馈表无状态）

### 决策 2：Insight 不注入 create（Phase 1-2）

**选择**：竞品报告只写入 Obsidian，人读后手动决策

**理由**：
- 小样本量（<50条）下的统计结论噪声太大
- insight 如果被当作硬约束，create 生成的内容会趋同失去创意
- Phase 3 等数据基础成熟后再谈自动化应用

### 决策 3：竞品报告不覆盖历史

**选择**：每次生成新报告存入 Obsidian 竞品库，不覆盖旧报告

**理由**：
- 竞品趋势变化比单次报告更有价值
- 每次新报告对比旧报告，能看到竞品规律的时间演变
- 文件名含日期：`竞品分析报告-2026-05-18.md`

## 6. 暂缓功能清单

| 功能 | 暂缓原因 |
|------|----------|
| 认知标签提取 | 依赖评论数据获取，当前无法落地 |
| 竞品认知机制分析 | 依赖竞品用户行为数据 |
| Insight 注入 create | 小样本统计结论不可信 |
| AI Agent 工作流 | 需要完整数据闭环后 |
| 反馈表自动提醒 | 飞书多维表格不支持定时提醒字段 |

---

## 7. 成功标准

| 指标 | 目标 |
|------|------|
| 反馈表数据录入率 | 发布后 7 日内录入比例 >80% |
| 竞品报告生成频率 | 每次竞品表有新 analyzed 记录时运行 |
| 竞品报告阅读率 | 报告生成后 24h 内人工阅读并决策 |
| 反馈记录样本量（Phase 2 门槛） | >50 条 before 开启认知标签分析 |
