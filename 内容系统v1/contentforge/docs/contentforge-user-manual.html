ContentForge 操作手册

一款基于 AI 的多平台内容生成工作流引擎，支持从关键词自动生成公众号、小红书、抖音文章，以及爆款文章的智能二创。

场景 A：原创生成

输入关键词，AI 自动分析主题并生成适配三个平台的原创文章

场景 B：爆款二创

粘贴一篇爆款文章，AI 自动解构爆款基因，生成差异化二创内容

竞品追踪

抓取微信公众号文章，分析竞品内容策略，提取可复用碎片

质量保障

内置原创度检测、合规审查、风格融合，确保输出质量

快速入门 · 5 分钟上手

在开始之前，请确保你的电脑安装了以下软件：

API Key 请前往对应平台官网申请：Kimi（yunwu.ai）、Tavily（tavily.com）

第四步：运行第一个内容生成

ContentForge 使用 skill 命令作为统一入口，用自然语言描述需求即可：

1

进入项目目录，运行 skill 命令：

cd contentforge
node dist/index.js skill "帮我写一篇关于AI的文章 发公众号"

命令格式：node dist/index.js skill "<需求描述>"
自动识别：原创生成 vs 爆款二创、目标平台（公众号/小红书/抖音）

两阶段协议（原创生成）

原创内容分两个阶段执行，确认大纲后再生成正文：

阶段命令做什么
Phase 1：出大纲skill "..." --phase outline生成大纲摘要，输出 runId
Phase 2：生成正文skill "..." --phase content --run-id <ID>确认大纲后生成完整文章
为什么分两阶段？
大纲阶段可以看到：推荐角度、标题候选、章节结构。确认后再生成，避免生成后才发现跑偏。你可以修改标题、注入真实案例/经历、调整案例方向，素材会进入 content-generation 的 prompt。

TUI 交互模式说明

ContentForge 会自动检测当前终端是否支持交互：

运行方式效果
直接运行命令自动进入 TUI 交互模式，提示确认后继续
加 --auto全自动运行，适合 CI / 批量 / 脚本调用
管道输入 / 非 TTY 环境自动退化为非交互模式，不会卡住
哪些命令支持 TUI 交互？
skill（大纲确认需交互）、recreate（差异化方向选择）、topic scrape（碎片提取确认）、style（风格选择）均支持 TUI 交互。

场景 A：原创内容生成

命令格式

node dist/index.js create --keyword "关键词"
node dist/index.js create --keyword "关键词" --no-interactive   # 全自动

工作流程（6 步 Pipeline）

Step 1 — 主题深挖  自动

输入关键词后，AI 自动发散出 10-15 个子话题、痛点、热门角度、争议话题、目标人群分析

Step 2 — 三平台选题分配  需确认

AI 为公众号、小红书、抖音分别选定差异化的切入角度（重合度不超过 30%）

Step 3 — 大纲生成（三路并行）  自动

根据各平台特点生成内容大纲：钩子设计、论点结构、案例位置、结尾升华

Step 4 — 素材检索（可选）  可选

通过搜索 API 检索相关数据、案例、故事素材

Step 5 — 全文生成（三路并行）  自动

按大纲逐段生成正文：公众号 2000-3000 字、小红书 500-800 字、抖音 150-300 字

Step 6 — 审校优化  自动

平台适配性审查 + 质量评分（标题吸引力/开头留存/内容价值/情绪调动/互动引导）+ 5 个候选标题

输出结果

运行完成后，产物保存在 output/<时间戳>/ 目录下：

output/
└── 2026-05-02_10-30-00_AI/
    ├── step1_topic-analysis.json      # 主题分析
    ├── step2_topic-assignments.json   # 三平台选题
    ├── step3_outlines.json            # 三平台大纲
    ├── step5_content-generation.json  # 生成的正文
    └── step6_review-optimization.json # 审校结果 + 终稿

场景 B：爆款二创

命令格式

node dist/index.js recreate --input ./article.md
node dist/index.js recreate --input ./article.md --direction auto      # 自动选方向
node dist/index.js recreate --input ./article.md --direction interactive # 交互选方向

工作流程（5 步 Pipeline）

Step 1 — 爆款解构分析  自动

深度解剖原文的爆款基因：击中的痛点/痒点、叙事结构、开头钩子技巧、情绪曲线、高传播力金句

Step 2 — 差异化方向生成  需确认

AI 生成 3-5 个差异化二创方向（视角迁移、受众迁移、内容替换策略），可选择自动或手动选定方向

Step 3 — 新大纲生成  自动

保留叙事骨架，用全新内容填充。⚠️ 关键约束：原文全文不传入后续步骤，从架构上防止抄袭

Step 4 — 全文生成  自动

按新大纲生成二创文章，Prompt 明确禁止复用原文句子、案例、比喻

Step 5 — 双重审查  自动

原创度审查（逐段对比，标记相似段落）+ 爆款潜力评估（与原文对比 5 个维度评分）。如有不达标段落，自动回到 Step 4 重写（最多循环 3 次）

内容进化的工作原理

系统通过三个阶段让创作能力不断提升：

阶段做什么数据来源
记录追踪每次改稿的决定revision manifest
分析统计什么写法受欢迎feedback records + competitor records
应用生成时参考学到的偏好buildPreferencePrompt()

竞品洞察注入

运行 learn --update-preferences 时，系统会：

  1. 从竞品表读取所有已分析记录，计算各叙事结构的平均互动率
  2. 找出竞品中高表现的结构和调性（样本量 > 0 时生效）
  3. 将竞品洞察写入 competitorInsights 区块
  4. 生成内容时，buildPreferencePrompt() 会输出竞品洞察段落作为参考

时间衰减权重

当反馈记录超过 30 条时，系统启用时间衰减：

weight = e^(-λ × daysAgo)，λ = 0.01（约 100 天衰减到 36%）

即近期数据权重更高，历史数据逐渐淡化。样本量低于 30 时不启用，保持简单平均。

常见问题（FAQ）

Q: 运行时报错 "Command not found"

确保在项目目录下运行命令，并使用完整路径：

node dist/index.js create --keyword "AI"

Q: 报 API Key 错误

检查 .env 文件是否放在项目根目录，以及 Key 是否正确。API Key 区分开发环境和生产环境。

Q: 交互模式下如何退出？

按 Ctrl + C 可随时退出当前操作。

Q: Step 5 素材检索可以关闭吗？

可以。关闭素材检索将跳过 Step 4，直接进入全文生成，可节省 API 调用成本。

Q: 如何查看详细日志？

运行命令时，默认会输出进度信息。如果需要调试日志，可以查看 output/<run-id>/ 目录下的中间产物 JSON 文件。

Q: 如何批量生成内容？

将要生成的关键词按行写入文件（如 keywords.txt），然后运行：

node dist/index.js batch --input ./keywords.txt --scenario create

Q: 内容生成中途失败了怎么办？

使用断点续跑功能，找到上次运行的 ID：

node dist/index.js resume --run-id <run-id> --from-step 3
