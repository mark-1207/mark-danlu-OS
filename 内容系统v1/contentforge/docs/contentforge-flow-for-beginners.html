ContentForge 小白入门指南

5 步完成从竞品分析到多平台内容发布的全流程

1
抓竞品 — 找到好文章
把网上写得好的文章抓进来，让 AI 学习为什么它火
🌐
复制文章链接
打开微信/小红书，看到好文章后复制链接
（点击文章右上角 → 复制链接）
✓ 不用做任何处理
→
⚡
粘贴给 ContentForge
在终端输入命令，把链接扔给它
它会自动抓取文章内容并存到飞书
终端命令
node dist/index.js topic scrape --url "粘贴的链接"
✓ 支持批量：一行一个链接
→
📊
飞书自动生成记录
抓取成功后，飞书表格里会自动出现一条记录
状态显示"待分析"
✓ 等 AI 分析完成（约几分钟）
2
AI 分析 — 找出爆款规律
让 AI 自动读懂文章好在哪：结构/调性/角度/标签
🤖
一键分析全部竞品
一条命令让 AI 逐篇分析飞书里所有待分析文章
自动提取：叙事结构、情感调性、内容角度、标签
终端命令
node dist/index.js learn --analyze
✓ 自动跳过已分析的文章
→
🧩
提取碎片进知识库
把好文章的精华部分（开头/金句/结尾）
自动拆解成原子卡片存入 Obsidian
终端命令
node dist/index.js learn --extract-fragments
✓ 写入 Obsidian 知识库，生成时随时调用
3
生成内容 — 写出自己的文章
告诉 ContentForge 你想写什么，它结合竞品洞察和你的知识库生成初稿
💬
用中文描述需求（Phase 1）
直接说："帮我写一篇关于职场加薪的公众号文章"
它自动理解意图，生成大纲供你确认
第一步：生成大纲
node dist/index.js skill "帮我写一篇关于职场加薪的文章 发公众号" --phase outline
✓ 也可以发给微信机器人处理
→
✅
确认大纲后再生成（Phase 2）
看到大纲后确认或修改标题/案例方向
注入你的真实经历，再生成完整文章
第二步：生成内容
node dist/index.js skill "帮我写一篇..." --phase content --run-id <ID>
✓ run-id 从上一步输出中复制
→
✨
AI 自动 review + 同步
自动检查质量分数，同步到 Obsidian
直接复制发布，或用 revise 继续改稿
继续改稿
node dist/index.js revise
✓ 支持直接编辑段落内容
4
发布 — 同步到各平台
一篇文章同时生成公众号/小红书/抖音三个版本
📝
生成多平台版本
创建时默认生成三个平台的内容
每个平台的结构/风格/长度都不同
✓ 公众号重深度，小红书重美观，抖音脚本
→
📤
复制到对应平台发布
文章自动存入 Obsidian
手动复制到对应平台发布即可
✓ 未来计划：直接对接发布 API
5
反馈 — 让系统越学越聪明
发布后把数据填回系统，它会自动分析什么写法更受欢迎
📈
7天后填入数据
到各平台后台复制：阅读量/点赞/评论/转发
用一条命令快速录入
快捷录入
node dist/index.js learn --feedback --run-id abc123 --likes 500 --comments 80 --shares 30 --reads 8000 --platform wechat
✓ run-id 在文章文件名里找
→
🧠
更新创作偏好
系统分析你的反馈 + 竞品数据
自动更新创作偏好（存在飞书）
终端命令
node dist/index.js learn --update-preferences
✓ 样本越多，偏好越准
→
👁
查看学到了什么
随时查看当前学到的创作偏好
知道什么结构/调性最适合你
终端命令
node dist/index.js learn --show-preferences
✓ 加 --json 输出原始数据
💡
从哪里开始？
先用 skill 命令生成第一篇文章，它会自动理解你的意图。整个流程分两步：先出大纲确认，再生成内容。用 --help 查看所有可用选项。
🔧
遇到问题怎么办？
每个命令都加 --help 查看详细说明。比如 node dist/index.js skill --help 查看 skill 命令的全部选项。
🔄
系统多久更新一次？
每次发布后填入反馈数据就跑一次 --update-preferences。样本超过 30 条后启用时间衰减，优先看近期数据。
