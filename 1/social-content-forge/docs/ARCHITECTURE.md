# Social Content Forge 架构文档

## 项目定位

**目标**：将任意输入转化为多平台高质量内容

**核心理念**：质量 > 数量。不是量产内容，而是创作能传播的爆款。

---

## 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户层                               │
│         (CLI / 对话式交互 + hook安全护栏)               │
├─────────────────────────────────────────────────────────┤
│                    输入层                               │
│    URL提取 │ 主题搜索 │ 素材解析                        │
├─────────────────────────────────────────────────────────┤
│                  内容理解层                             │
│    原子块拆分 │ 解码报告 │ 病毒元素识别                  │
├─────────────────────────────────────────────────────────┤
│                  质量评估层                             │
│    六维度评分 │ 决策路径 │ 诊断报告 │ 爆款预测           │
├─────────────────────────────────────────────────────────┤
│                  平台适配层                             │
│    微信适配器 │ 小红书适配器 │ Twitter适配器             │

│                    输出层                               │
│       文件(.md) │ SQLite │ 飞书多维表格                │
└─────────────────────────────────────────────────────────┘
```

## 核心数据模型

### ContentAtom（原子内容块）

内容拆解的最小单位，可跨平台复用：

```typescript
interface ContentAtom {
  id: string;
  type: '观点' | '金句' | '案例' | '数据' | '故事' | '方法论';
  content: string;
  viralElements: string[];
  platformSuitability: Platform[];
  reusability: 'high' | 'medium' | 'low';
}
```

### EvaluationResult（评估结果）

```typescript
interface EvaluationResult {
  overallScore: number;        // 0-100
  dimensionScores: {
    emotion: number;          // 0-10
    utility: number;         // 0-10
    narrative: number;        // 0-10
    socialCurrency: number;  // 0-10
    controversy: number;     // 0-10
    timeliness: number;       // 0-10
  };
  decisionPath: 'A' | 'B' | 'C';
  diagnostics: Diagnostic[];
  viralPrediction: {
    platform: string;
    readRange: string;
    confidence: number;
  }[];
}
```

## 质量评估体系

基于病毒传播学研究，六维度评分：

| 维度 | 权重 | 理论依据 |
|------|------|---------|
| 情绪激发度 | 25% | "情绪唤醒"是转发第一驱动力 |
| 实用价值 | 25% | 实用内容被收藏=长期价值流量 |
| 叙事结构 | 20% | 故事型内容传播率比干条高30倍 |
| 社交货币 | 15% | 转发塑造社交形象 |
| 争议引导 | 10% | 争议内容评论率显著更高 |
| 时效贴切 | 5% | 热点加持，但不是决定因素 |

**决策阈值**：
- ≥80分：直接适配
- 60-79分：优化后适配
- <60分：建议重构

## LLM路由策略

| 任务 | 模型 | 原因 |
|------|------|------|
| 质量评估 | Claude/GPT | 判断力要求高 |
| 微信公众号 | Claude/GPT | 深度长文，质量优先 |
| 小红书 | Claude/GPT | 情绪共鸣要求高 |
| Twitter | deepseek/kimi | 短内容，速度优先 |

## 平台适配要点

### 微信公众号
- 标题：8-30字，有钩子
- 结构：开头钩子 → 方法论 → 案例 → 金句 → 结尾行动号召
- 字数：2000字左右

### 小红书
- 封面文字：吸睛、有悬念
- 标签：3-5个话题标签
- 文风：情绪强、个人体验感、种草感
- 字数：800字左右

### Twitter
- 观点鲜明、立场明确
- 字数：≤280字
- 结构：可扩展为Thread
- 钩子：前3字必须抓人

## 扩展性设计

### 新增平台适配器

1. 在 `src/adapters/` 下创建新目录
2. 实现 `PlatformAdapter` 接口
3. 在 `src/llm/router.ts` 注册路由
4. 更新 SKILL.md 的平台列表

### 新增数据源

1. 在 `src/extractor/` 添加提取器
2. 识别新URL类型并路由
3. 更新输入类型枚举
