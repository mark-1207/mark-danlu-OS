# Evaluator 模块

内容质量评估器。负责对内容进行六维度评分、决策路径判断、诊断报告生成。

## 核心职责

1. 接收 analyzer 输出的原子内容块和解码报告
2. 进行六维度评分
3. 输出决策路径建议
4. 生成诊断报告和优化建议
5. 给出爆款预测

## 六维度评分标准

| 维度 | 权重 | 评分逻辑 |
|------|------|---------|
| 情绪激发度 | 25% | 评估情绪类型(焦虑/兴奋/愤怒/共鸣/好奇)和强度 |
| 实用价值 | 25% | 评估干货密度：方法/工具/数据/案例的具体程度 |
| 叙事结构 | 20% | 评估故事弧线完整性和叙事流畅度 |
| 社交货币 | 15% | 评估转发能彰显的读者身份/品味/立场 |
| 争议引导 | 10% | 评估观点开放性和讨论空间 |
| 时效贴切 | 5% | 评估与当前热点/趋势的关联度 |

## 评分计算

```
总分 = Σ(维度得分 × 权重) × 10
输出范围：0-100
```

## 决策路径

| 评分 | 路径 | 操作 |
|------|------|------|
| ≥80 | A | 具备爆款潜力，直接进入平台适配 |
| 60-79 | B | 可发布但建议优化，输出诊断后继续 |
| <60 | C | 缺乏爆款基因，建议重构后重新评估 |

## 诊断报告格式

每个维度输出：
```
{
  dimension: "情绪激发度",
  score: 7,
  issue: "开头场景不够具体，情绪张力不足",
  suggestions: [
    "把'很多人觉得AI难'改为'我用Claude 3天完成了以前3周的活'",
    "在方法论之间加入'踩坑经历'制造共鸣"
  ]
}
```

## 爆款预测

针对每个目标平台输出：
```
{
  platform: "微信",
  readRange: "3万-8万",
  confidence: 0.75,
  factors: ["情绪6分+实用8分+叙事5分", "标题有数字"],
  risks: ["开头钩子不够强，可能在第3段流失读者"]
}
```

## LLM调用策略

- 使用 Claude/GPT 进行评分（需要强判断力）
- 如果评分<60，必须给出具体重构建议，不能只说"质量不足"
- 评分时要求输出评分理由，便于用户理解和校准

## 依赖

- 输入：analyzer输出的 atoms + 解码报告
- 输出：evaluation报告（JSON + markdown格式）
- 依赖外部：Claude/GPT API
