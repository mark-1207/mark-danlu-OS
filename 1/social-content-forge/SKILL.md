# Social Content Forge

## 项目状态: MVP 完成 ✅

社媒内容多平台生成流水线：将任意输入转化为微信/小红书/Twitter高质量内容，全自动同步飞书多维表格。

## 核心原则

- **质量 > 数量**：每篇内容必须具备情绪共鸣+实用价值+故事叙事
- **先跑通，再扩展**：MVP只做核心流程，不做多余功能
- **爆款导向**：不是创作内容，而是创作能传播的内容

## 执行命令

```bash
cd social-content-forge

# 运行CLI（默认同步飞书）
npm run cli -- "你的主题"

# 禁用飞书同步
npm run cli -- --no-sync-feishu "你的主题"

# 开发模式
npm run dev
```

## 质量评估标准

| 维度 | 权重 | 目标分数 |
|------|------|---------|
| 情绪激发度 | 25% | ≥7 |
| 实用价值 | 25% | ≥8 |
| 叙事结构 | 20% | ≥6 |
| 社交货币 | 15% | ≥6 |
| 争议引导 | 10% | ≥5 |
| 时效贴切 | 5% | ≥4 |

**总分≥80** → 直接适配
**总分60-79** → 优化后适配
**总分<60** → 建议重构

## 平台优先级

1. 微信公众号（核心，深度长文）
2. 小红书（图文，情绪强）
3. Twitter（短文，观点鲜明）

## 规则

1. 所有API凭证必须从环境变量读取，禁止硬编码
2. 评分<60的内容必须输出诊断报告，不能直接跳过
3. 飞书同步失败必须记录日志并降级，不能静默忽略
4. 每个平台适配后必须自检清单
